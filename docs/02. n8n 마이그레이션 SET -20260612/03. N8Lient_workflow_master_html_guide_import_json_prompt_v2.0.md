# N8Lient 워크플로우 마스터 등록 HTML 가이드 & Import JSON 생성 프롬프트 v2.0 단독형

마이그레이션이 완료된 **최종 n8n 워크플로우 JSON**을 기준으로, 이 워크플로우를 N8Lient 앱의 **N8N 워크플로우 마스터 등록 화면**에 입력하기 위한 등록 가이드를 생성하라.

이 프롬프트는 **단독 사용**을 전제로 한다.  
기본적으로는 아래 2개만 첨부되어 있어도 작업을 진행해야 한다.

```text
1. 이 프롬프트 파일
2. 최종 n8n 워크플로우 JSON 파일
```

N8Lient 기초설계서, 마이그레이션 지시서, 테스트 결과가 함께 첨부되어 있으면 참고하되, 첨부되어 있지 않더라도 추가 자료를 요구하지 말고 최종 n8n JSON에서 확인 가능한 값만 사용한다.

---

## 0. 최종 산출물 목표

결과물은 **단일 HTML 문서**로 작성한다.

이 HTML은 두 가지 목적을 동시에 가진다.

```text
1. 오퍼레이터가 보면서 N8Lient 워크플로우 마스터 등록값을 확인·수정할 수 있는 A4 출력용 가이드
2. HTML 안에서 수정한 값을 N8Lient 앱에 업로드 가능한 표준 Import JSON으로 다운로드하는 도구
```

즉, HTML은 단순 정적 가이드가 아니라 **편집 가능한 체크리스트형 등록 가이드**여야 한다.

---

## 1. 핵심 원칙

1. n8n 원본 JSON을 수정하지 않는다.
2. N8Lient 앱이 직접 raw n8n JSON을 분석하지 않는다는 전제를 따른다.
3. 이 HTML은 n8n JSON을 사람이 검토 가능한 등록 가이드로 풀어주고, 최종적으로 **N8Lient 표준 Import JSON**을 생성하는 중간 산출물이다.
4. 확실하지 않은 값은 억지로 확정하지 말고 `확인 필요`로 표시한다.
5. 단, Import JSON으로 다운로드될 값에는 가능하면 `확인 필요` 같은 문구를 그대로 넣지 말고, 빈 값 또는 안전한 기본값을 넣은 뒤 HTML에서 경고로 표시한다.
6. Credential, Token, Secret, API Key, Access Token, Refresh Token, Private Key, Service Account 값은 절대 출력하지 않는다.
7. Google Drive Optional Export가 확인되면 최신 기준인 **MD 파일 보관 폴더**와 **첨부파일 보관 폴더**를 분리하는 구조를 우선 제안한다.
8. HTML은 외부 라이브러리 없이 단일 파일로 동작해야 한다.
9. HTML은 A4 출력 시 그대로 보기 좋은 체크리스트 문서 형태여야 한다.
10. 불필요하게 화려한 대시보드 스타일을 쓰지 말고, 실무 등록 체크리스트처럼 단순하게 만든다.

## 2. 입력 자료 우선순위

입력 자료는 아래 우선순위로 해석한다.

```text
1순위: 최종 n8n 워크플로우 JSON
2순위: 마이그레이션 완료 보고서 또는 테스트 결과
3순위: N8Lient 기초설계서 / 마이그레이션 지시서
```

단, 2순위와 3순위 자료가 없어도 작업을 중단하지 않는다.

추가 자료 요청은 하지 않는다.

## 3. 출력 형식

반드시 완전한 HTML 문서로 출력하라.

```html
<!doctype html>
<html lang="ko">
<head>
  <meta charset="utf-8">
  <title>...</title>
  <style>...</style>
</head>
<body>
  ...
  <script>...</script>
</body>
</html>
```

HTML 요구사항:

```text
CSS는 내부 <style>에 작성
JavaScript는 내부 <script>에 작성
외부 라이브러리 사용 금지
모든 주요 값은 input/select/textarea/checkbox로 수정 가능
복사 버튼 제공
N8Lient 표준 Import JSON 다운로드 버튼 제공
A4 출력 버튼 제공
모바일 대응은 하되, 최우선은 A4 출력 가독성
```

## 7. 필드 상태 색상

각 입력 필드는 상태를 가진다.


색상은 과하지 않게 사용한다.

```css
.field.ok { border-left: 4px solid #3b82f6; background: #eff6ff; }
.field.warning { border-left: 4px solid #f97316; background: #fff7ed; }
.field.error { border-left: 4px solid #ef4444; background: #fef2f2; }
```

각 필드에는 짧은 가이드 문구를 붙인다.

```text
예:
- 정상: 저장 가능한 값입니다.
- 경고: 자동 추정값입니다. 저장 전 확인하세요.
- 오류: 필수값입니다. 저장 전 입력해야 합니다.
```

## 9. Import JSON 다운로드 기능

HTML 안에서 수정한 값을 기반으로 아래 JSON 파일을 다운로드할 수 있어야 한다.

다운로드 JSON은 반드시 N8Lient 앱에서 불러올 수 있는 표준 Import JSON 형태여야 한다.

```json
{
  "schemaVersion": "n8lient.workflowTemplateImport.v1",
  "workflowTemplate": {
    "workflowKey": "",
    "name": "",
    "shortName": "",
    "version": "1.0.0",
    "status": "draft",
    "description": "",
    "webhookSecretId": "",
    "n8nServerKey": "main",
    "inputSchema": {
      "titleRequired": true,
      "acceptedInputTypes": [],
      "allowedExtensions": [],
      "maxFileSizeMB": 10
    },
    "retentionCapabilities": {
      "maxLevel": "processed_result",
      "defaultLevel": "processed_result",
      "supportedLevels": ["notify_only", "processed_result"],
      "supportsProcessorResult": true,
      "supportsOriginalFileRefs": false,
      "supportsResultRefs": false,
      "supportsResultPolicyRouter": false
    },
    "operatorRetentionPolicy": {
      "allowedLevels": ["notify_only", "processed_result"],
      "defaultLevel": "processed_result",
      "allowCompanyOverride": true,
      "allowUserOverride": true
    },
    "configSchema": []
  }
}
```

다운로드 파일명 규칙:

```text
N8Lient_워크플로우_마스터_ImportJSON_{workflowKey}_v1.0.json
```

workflowKey가 비어 있으면:

```text
N8Lient_워크플로우_마스터_ImportJSON_확인필요_v1.0.json
```

## 10. Import JSON 생성 스크립트 요구사항

HTML의 JavaScript는 화면 값을 읽어 Import JSON을 생성해야 한다.

필수 함수:

```text
collectImportJson()
downloadImportJson()
copyToClipboard()
printGuide()
```

`collectImportJson()`은 HTML 폼의 현재 값을 읽어 아래 객체를 만든다.

```js
{
  schemaVersion: "n8lient.workflowTemplateImport.v1",
  workflowTemplate: { ... }
}
```

배열 처리 기준:

```text
acceptedInputTypes: 체크된 값만 배열로 생성
allowedExtensions: 쉼표 문자열을 배열로 변환
supportedLevels: 체크된 보관 레벨만 배열로 생성
operatorRetentionPolicy.allowedLevels: 체크된 허용 레벨만 배열로 생성
configSchema: 화면의 설정 필드 카드 순서대로 배열 생성
select options: 쉼표 문자열을 배열로 변환
```

boolean 처리 기준:

```text
checkbox checked => true
checkbox unchecked => false
```

number 처리 기준:

```text
maxFileSizeMB, number type 값은 Number()로 변환
NaN이면 기본값 또는 0이 아닌 안전값 사용
```

## 11. 섹션 1. 워크플로우 기본 정보

아래 항목을 편집 가능한 필드로 출력한다.

```text
워크플로우 Key
워크플로우 이름
줄임말
버전
배포 상태
Webhook Secret 참조
n8n 서버 식별 Key
설명
```

필드 기준:

### 워크플로우 Key

```text
data-path: workflowTemplate.workflowKey
형식: 영문 소문자, 숫자, 하이픈
상태:
- 형식과 값이 확실하면 ok
- 자동 추정이면 warning
- 비어 있으면 error
```

### 워크플로우 이름

```text
data-path: workflowTemplate.name
사용자가 이해하기 쉬운 한글명
```

### 줄임말

```text
data-path: workflowTemplate.shortName
2~4자 권장
```

### 버전

```text
data-path: workflowTemplate.version
기본값: 1.0.0
```

### 배포 상태

```text
data-path: workflowTemplate.status
select: draft, published
테스트 전이면 draft
```

### Webhook Secret 참조

```text
data-path: workflowTemplate.webhookSecretId
기본적으로 workflowKey와 동일하게 권장
실제 Secret 값이 아니라 참조 ID만 사용
```

### n8n 서버 식별 Key

```text
data-path: workflowTemplate.n8nServerKey
기본값: main
```

### 설명

```text
data-path: workflowTemplate.description
1~2문장
```

## 12. 섹션 2. inputSchema

아래 항목을 편집 가능한 필드로 출력한다.

```text
실행 제목 필수 여부
허용 입력 형태
허용 파일 확장자
최대 파일 크기(MB)
```

### 실행 제목 필수 여부

```text
data-path: workflowTemplate.inputSchema.titleRequired
checkbox
```

### 허용 입력 형태

체크박스 형태로 출력한다.

```text
text
file
image
audio
```

n8n JSON에서 실제 필요한 것만 체크한다.

중요:

```text
file_0 같은 generic 이름이 있다고 해서 file을 자동 체크하지 않는다.
audio binary가 핵심이면 audio를 체크한다.
image 처리가 명확하지 않으면 image는 체크하지 않는다.
문서 파싱 노드가 명확하지 않으면 file은 체크하지 않는다.
```

### 허용 파일 확장자

```text
data-path: workflowTemplate.inputSchema.allowedExtensions
쉼표 문자열로 표시
예: mp3, webm, m4a, wav
```

### 최대 파일 크기

```text
data-path: workflowTemplate.inputSchema.maxFileSizeMB
숫자
```

확실하지 않으면 권장 기본값을 넣고 warning으로 표시한다.

```text
workflowTemplate.retentionCapabilities.supportsOriginalFileRefs
workflowTemplate.retentionCapabilities.supportsResultRefs
workflowTemplate.retentionCapabilities.supportsResultPolicyRouter
```

## 15. 섹션 5. configSchema

configSchema는 실제 화면의 설정 필드 반복 구조에 맞춘다.

각 설정 필드는 아래 값을 편집 가능하게 출력한다.

```text
설정 Key
라벨 이름
인풋 타입
기본값 출처
필수 입력 항목 여부
입력 힌트
가이드 설명
select options
```

JSON export 기준:

```json
{
  "key": "",
  "label": "",
  "inputType": "text",
  "defaultSource": "직접 입력",
  "required": false,
  "placeholder": "",
  "description": "",
  "options": []
}
```

인풋 타입은 아래 중 하나만 사용한다.

```text
text
textarea
email
number
boolean
select
password
url
```

`select` 타입이면 options를 반드시 쉼표 문자열로 출력하고, export 시 배열로 변환한다.

## 18. Google Drive Optional Export 표준

워크플로우가 Google Drive Optional Export를 지원하거나 구형 `googleDriveExportFolderId`가 감지되면, 아래 표준 5개 필드를 제안한다.

```text
optionalExportProvider
googleDriveMdFolderName
googleDriveMdFolderId
googleDriveAttachmentFolderName
googleDriveAttachmentFolderId
```

구형 `googleDriveExportFolderId`는 그대로 export하지 않는다.

확인 필요 항목에 아래 문구를 넣는다.

```text
구형 googleDriveExportFolderId가 감지되었습니다. 최신 기준은 MD 폴더와 첨부파일 폴더를 분리합니다. n8n 워크플로우도 이 구조에 맞게 수정되어 있는지 확인하세요.
```

## 20. 복사 버튼 구현

모든 주요 입력값은 복사 가능해야 한다.

```html
<button type="button" data-copy-target="fieldId">복사</button>
```

스크립트 예시:

```js
document.querySelectorAll("[data-copy-target]").forEach((button) => {
  button.addEventListener("click", async () => {
    const target = document.getElementById(button.dataset.copyTarget);
    const text = target ? ("value" in target ? target.value : target.innerText).trim() : "";
    await navigator.clipboard.writeText(text);
    const old = button.innerText;
    button.innerText = "복사됨";
    setTimeout(() => button.innerText = old, 1000);
  });
});
```
