첨부한 보존형 기초설계서 문서세트를 최상위 기준으로 삼아라.
특히 문서세트에 `07_엔팔라이언트_결과_DB화_지식검색_최소_계약`이 포함되어 있다면, Level 2 이상 결과 DB화/검색 최소 구조도 함께 따른다.

첨부한 n8n 워크플로우 JSON을 엔팔라이언트 표준으로 마이그레이션하려고 한다.

중요:

- 기존 정상 동작 중인 업무 처리 로직은 최대한 보존한다.
- 수정 전 먼저 변경 대상 노드와 이유를 보고하라.
- 승인 전에는 JSON을 수정하거나 새 JSON을 생성하지 마라.
- 문서 간 충돌이 있으면 보존형 기초설계서의 최신 정책을 우선한다.

첨부문서 :

1. 기초설계서 문서세트 zip
2. 수정 대상 n8n 워크플로우 JSON
3. 아래 마이그레이션 프롬프트


기초설계서를 아직 확인 못했다면 "기초 설계문서를 모두 주세요" 라고 말하라
기초 설계서를 모두 확인 했다면 "마이그레이션 지시 프롬프트를 주세요" 라고 말 하라.
마이그레이션 지시 프롬프트까지 확인을 마쳤다면 "마이그레이션 대상 json 파일을 주세요" 라고 말하라

주의:
- Level 1 notify_only는 이메일/알림 중심이므로 자유 포맷을 허용한다.
- Level 2 processed_result 이상부터는 processorResult 최소 공통 구조를 맞춘다.
- 벡터라이징, 백링크, 지식 그래프, 통합 검색 인덱스 생성은 n8n 실행 단계에서 하지 않는다.
