첨부한 최신 N8Lient 기초설계서 문서세트를 최상위 기준으로 삼아라.
보존형 문서와 리뉴얼 문서가 함께 제공되면, 리뉴얼 문서와 최신 정책을 우선한다.
특히 문서세트에 `07_엔팔라이언트_결과_DB화_지식검색_계약`이 포함되어 있다면, Level 2 이상 결과 DB화/검색 최소 구조도 함께 따른다.
또한 문서세트에 `08_엔팔라이언트_Google_Drive_Optional_Export_계약`이 포함되어 있다면, Google Drive Optional Export는 MD 파일 보관 폴더와 첨부파일 보관 폴더를 분리하는 최신 기준을 따른다.

첨부한 n8n 워크플로우 JSON을 엔팔라이언트 표준으로 마이그레이션하려고 한다.

중요:

- 기존 정상 동작 중인 업무 처리 로직은 최대한 보존한다.
- 수정 전 먼저 변경 대상 노드와 이유를 보고하라.
- 승인 전에는 JSON을 수정하거나 새 JSON을 생성하지 마라.
- 문서 간 충돌이 있으면 최신 N8Lient 기초설계서의 현재 정책을 우선한다.

첨부문서:

1. 최신 기초설계서 문서세트 zip
2. 수정 대상 n8n 워크플로우 JSON
3. 아래 마이그레이션 프롬프트

기초설계서를 아직 확인 못했다면 "기초 설계문서를 모두 주세요" 라고 말하라.
기초설계서를 모두 확인했다면 "마이그레이션 지시 프롬프트를 주세요" 라고 말하라.
마이그레이션 지시 프롬프트까지 확인을 마쳤다면 "마이그레이션 대상 json 파일을 주세요" 라고 말하라.

주의:

- Level 1 `notify_only`는 이메일/알림 중심 경량형이다.
- `notify_only`에서도 결과를 Markdown 파일로 임시 생성하여 이메일에 첨부할 수 있다.
- 단, `notify_only`의 MD 첨부파일은 이메일 전송용 임시 결과물이며, N8Lient DB/Storage/resultRefs에 보관하지 않는다.
- Level 2 `processed_result` 이상부터는 `processorResult` 최소 공통 구조를 맞추고, DB 저장/검색 대상으로 삼을 수 있다.
- Level 3 `full_archive`는 Level 2에 더해 원본/첨부파일과 결과 파일 참조를 Storage와 연결한다.
- 벡터라이징, 백링크, 지식 그래프, 통합 검색 인덱스 생성은 n8n 실행 단계에서 하지 않는다.
- Google Drive Optional Export가 필요한 경우, 단일 Google Drive 폴더 ID 하나로 MD와 첨부파일을 섞지 않는다.
- Google Drive Optional Export 표준 설정 키는 `optionalExportProvider`, `googleDriveMdFolderName`, `googleDriveMdFolderId`, `googleDriveAttachmentFolderName`, `googleDriveAttachmentFolderId`다.
