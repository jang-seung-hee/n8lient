# N8Lient 커스터마이징 프롬프트 세트 — 처리 결과 확인 최소 보강 변경요약

- 생성일: 2026-06-22
- 작업 원칙: 기존 문서 삭제/통합 없이, 필요한 보강 섹션만 추가
- 대상: 작업모드 선택, Mode 1, Mode 2, Mode 3, Mode 4, Mode 99 프롬프트

## 공통 반영 원칙

```text
n8n 워크플로우 = 처리 결과 확인 메시지 작성
Gateway = callback 수신 및 submissions 저장
N8Lient 앱 = 결과 상세 화면의 [02] 처리 결과 확인 섹션에 표시
```

앱은 워크플로우별 완료 문구를 임의 생성하지 않는다. 워크플로우는 callback payload의 `result.summary`, `result.resultUrl`, `processorResult.summary`, `processorResult.structuredData.actionLinks`를 통해 사용자용 결과 확인 내용을 전달한다.

## 파일별 보강 내용

| 파일 | 보강 내용 |
|---|---|
| 01_START | 처리 결과 확인 메시지 설계가 필요한 작업모드 안내 추가 |
| 02A_MODE1 | 처리 결과 확인 메시지 초안 작성, 사용자 승인, [24A] Build User Result Message 노드 기준 추가 |
| 02B_MODE2 | Mode 1과 동일한 커스터마이징 기준 추가 |
| 02C_MODE3 | 이메일 디자인 Only에서는 callback/processorResult 변경 금지, 필요 시 Mode 1/2 전환 기준 추가 |
| 03_MODE4 | 등록 가이드 확인 필요 항목에 처리 결과 확인 메시지/링크 검토 기준 추가 |
| 99_MODE99 | 처리 결과 확인 누락/링크 미표시/actionLinks 오표시 디버깅 기준 추가 |

## 보강 범위

- 기존 원문 삭제 없음
- 기존 섹션 통합 없음
- 기존 금지/허용 범위 유지
- 처리 결과 확인 관련 최소 보강 섹션만 문서 말미에 추가

## n8n callback 권장 필드

```json
{
  "result": {
    "summary": "사용자에게 보여줄 짧은 처리 결과 메시지",
    "resultUrl": "사용자가 열 수 있는 대표 결과 URL"
  },
  "processorResult": {
    "summary": "[02] 처리 결과 확인 섹션에 표시할 사용자용 요약",
    "structuredData": {
      "actionLinks": [
        {
          "label": "사용자에게 표시할 링크 버튼 문구",
          "url": "사용자가 열 수 있는 URL",
          "type": "primary"
        }
      ]
    }
  }
}
```
