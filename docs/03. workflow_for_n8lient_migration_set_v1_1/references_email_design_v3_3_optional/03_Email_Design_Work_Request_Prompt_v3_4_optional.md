아래 파일들을 함께 참고하여 n8n 워크플로우의 이메일 리포팅 디자인을 수정하라.

## 참고 파일
1. `01_Email_Frame_Template_v3_4_optional_dansoft_footer.html`
2. `02_Email_Report_Design_v3_4_optional.md`
3. `04_Content_Pattern_Examples_v3_4_optional.html`
4. 실제 n8n 워크플로우 JSON 또는 Code 노드 소스

## 작업 목표
- 기존 워크플로우의 데이터 처리 로직은 변경하지 않는다.
- 기존 이메일 본문에 들어가던 정보와 의미는 유지한다.
- 공통 헤더, 상태바, 푸터는 프레임 템플릿을 적용한다.
- 본문은 `02_Email_Report_Design_v3_4_optional.md`의 원칙과 `04_Content_Pattern_Examples_v3_4_optional.html`의 실제 HTML 패턴 기준으로 문서형 리포트 구조로 재구성한다.

## 디자인 사용 제한 규칙
- 본문 디자인은 반드시 `04_Content_Pattern_Examples_v3_4_optional.html` 파일에 실제로 존재하는 HTML 패턴만 사용한다.
- 샘플 파일에 없는 레이아웃, 박스, 카드, 테이블, 컬럼, 장식 요소를 새로 만들지 않는다.
- `02_Email_Report_Design_v3_4_optional.md`는 디자인 원칙 문서이고, 실제 사용 가능한 HTML 구조의 최종 기준은 `04_Content_Pattern_Examples_v3_4_optional.html`이다.
- 필요한 정보가 많더라도 새로운 표나 박스 구조를 임의로 만들지 말고, 샘플에 있는 `섹션 표제`, `항목 제목`, `메타 정보`, `본문 역할 라벨`, `순번 목록`, `블릿 목록`, `태그`, `상태 배지`, `텍스트 링크`, `구분선`, `수평바` 패턴만 조합한다.
- `04_Content_Pattern_Examples_v3_4_optional.html`에 없는 구조를 사용해야 할 것 같으면, 새 구조를 만들지 말고 기존 텍스트/목록 구조로 풀어서 작성한다.

## 절대 금지
본문 `{{CONTENT_HTML}}` 내부에서는 아래 요소를 절대 생성하지 않는다.

- `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>`
- 표처럼 보이는 2열/3열 구조
- 카드 박스
- 그림자 박스
- 둥근 테두리 박스
- 넓은 배경색 박스
- 멀티컬럼 레이아웃
- URL 전체 문자열 노출
- 장식용 아이콘 또는 이모지

주의:
- 프레임 템플릿 내부의 이메일 호환용 `<table>` 구조는 그대로 둔다.
- 금지 대상은 AI가 생성하거나 수정하는 본문 `{{CONTENT_HTML}}` 내부에 적용한다.

## 표 데이터 처리 방식
기존 본문에 표 형태의 데이터가 있거나 표로 만들고 싶은 정보가 있어도 표를 만들지 않는다.

대신 다음 방식으로 변환한다.
- 숫자 비교: 순번 목록 또는 블릿 목록
- 설정값: `항목: 값` 형태의 메타 정보 또는 블릿 목록
- 오류 로그: 발생 시각 / 워크플로우명 / 오류 메시지 / 영향 범위 순서의 문단과 목록
- 정산/수치 정보: 항목별 한 줄 목록
- 처리 결과: 상태 배지 + 짧은 문단 + 목록

## 작업 방식
1. 실제 n8n 워크플로우 JSON 또는 Code 노드 소스를 먼저 확인한다.
2. 이메일 본문을 생성하는 노드만 식별한다.
3. 데이터 처리, 분기, API 호출, 저장 로직은 변경하지 않는다.
4. 이메일 HTML 프레임과 본문 디자인만 수정한다.
5. 본문 구조를 만들 때는 먼저 어떤 샘플 패턴을 사용할지 판단한 뒤, 그 패턴의 스타일만 재사용한다.
6. 샘플에 없는 스타일을 임의로 추가하지 않는다.
7. 수정 후 JSON 문법 오류가 없는지 검증한다.

## 출력
- 실제 n8n에 바로 임포트하거나 붙여넣을 수 있는 JSON 파일을 생성한다.
- 워크플로우 버전을 0.1 올린다.
- 수정된 JSON 파일과 필요한 참고 파일을 ZIP 파일로 제공한다.
- 변경 요약에는 수정한 노드명, 변경 전/후 요지, 디자인 변경 범위를 간단히 정리한다.
