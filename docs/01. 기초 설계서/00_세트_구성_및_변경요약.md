## 2. 이번 보강의 결론

기존 설계문서 전체를 갈아엎지 않고, 결과 데이터 권한관리 계약을 11번 신규 문서로 추가한다.

기존 문서 중 직접 영향이 있는 4개 문서만 최소 보강한다.

```text
01 솔루션 헌법
- 결과 보관 정책과 결과 열람 권한은 별도 정책이라는 최상위 원칙 추가

02 아키텍처 명세서
- accessMode 기준 결과 조회 아키텍처와 역할별 해석 추가

04 DB/Storage 계약
- submissions.accessMode 저장 필드와 접근 기준 추가

07 결과 DB화/지식검색 계약
- private/company 기준 검색·공유 정책 추가
```

나머지 문서는 기존 계약을 유지한다.

```text
03 결과/보관 레벨 계약
05 Gateway ↔ n8n 실행 계약
06 n8n 프로세서 전환 체크리스트
08 Google Drive Optional Export 계약
09 워크플로우 버전관리 운영 계약
10 Payload / Meta 표준 규약
```

## 3. 핵심 정책

```ts
accessMode: "private" | "company";
```

```text
private
- 개인 보관
- 작성자 본인만 결과 본문을 볼 수 있음
- 작성자가 허용된 워크플로우에서 회사 공개 가능
- 작성자 또는 관리자가 공개 취소/철회 가능

company
- 회사 공개
- 같은 회사 구성원이 결과 본문을 볼 수 있음
- 작성자는 임의로 비공개 전환 불가
- 관리자만 예외적으로 공개 철회 가능
```

---

## 4. n8n 영향 없음

이번 보강은 n8n workflow 또는 callback payload 구조 변경이 아니다.

```text
n8n 역할
- 자동화 실행
- 결과 생성
- Gateway callback 전달

N8Lient 역할
- 결과 저장
- accessMode 결정
- 결과 조회 권한 판단
- 작성자 공개/취소 처리
- 관리자 공개 철회 처리
```

## 5. 향후 확장

1차 구현은 `private/company`만 지원한다.

향후 필요하면 아래처럼 확장한다.

```ts
accessMode: "private" | "company" | "group" | "level" | "custom";
```

단, 그룹/보안레벨/특정 사용자 공유는 1차 구현 범위가 아니다.
