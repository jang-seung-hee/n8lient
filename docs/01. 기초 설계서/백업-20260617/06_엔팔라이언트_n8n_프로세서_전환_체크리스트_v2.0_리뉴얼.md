# 엔팔라이언트 n8n 프로세서 전환 체크리스트 v2.0

- 문서명: 엔팔라이언트 n8n 프로세서 전환 체크리스트
- 버전: v2.0 리뉴얼
- 작성일: 2026-06-16
- 문서 상태: 현행 전환 기준 문서
- 상위 문서: Gateway ↔ n8n 실행 계약 v2.0

---

## 1. 목적

기존 개인용 또는 단일 고객용 n8n 워크플로우를 엔팔라이언트용 프로세서로 전환할 때 확인해야 할 최소 기준을 정의한다.

전환 목표는 다음과 같다.

```text
n8n은 업무 처리에 집중한다.
권한, 설정 병합, 보관 정책 계산은 Gateway가 담당한다.
결과는 Gateway callback으로 반환한다.
```

---

## 2. 전환 전 필수 확인

```text
워크플로우 목적
입력 타입(text/file/image/audio)
필수 입력 조건
파일 크기/확장자 제한
결과 형태
이메일 보고 여부
원본 파일 보관 필요 여부
Optional Export 필요 여부
callback 가능 여부
```

---

## 3. 제거해야 할 구형 구조

아래 구조는 엔팔라이언트용 표준이 아니다.

```text
Cloud Memo 전용 payload 의존
개인 Google Drive 폴더 하드코딩
Firestore 직접 조회/수정
사용자별 Credential 하드코딩
Google Drive를 기본 저장소로 전제
input.title을 무조건 필수로 가정
모든 원본 파일을 무조건 저장
callback 없이 이메일만 보내고 종료
```

---

## 4. Gateway payload 수신

n8n Webhook은 Gateway가 보낸 canonicalPayload를 기준으로 처리한다.

필수 확인:

```text
submissionId
clientId
uid
workflowKey
automationId
callbackUrl
input
settings
retentionPolicy
files
```

n8n은 `payload.settings`만 사용한다. 회사/개인 설정을 직접 조회하지 않는다.

---

## 5. Header Auth 검증

Gateway → n8n 호출은 Header Auth로 검증한다.

```text
Header Name: X-N8N-TOKEN
```

n8n → Gateway callback도 Header Auth로 검증한다.

```text
Authorization: Bearer {N8N_CALLBACK_SECRET}
```

Secret 값은 노드 코드, 보고서, 로그에 노출하지 않는다.

---

## 6. title 계약 체크

n8n은 다음 title 계약을 지켜야 한다.

```text
input.title = 사용자가 직접 입력한 제목. 없으면 null.
titleProvided=false이면 input.title을 결과 제목으로 사용하지 않는다.
submissionTitle은 시스템 임시 표시 제목이다.
processorResult.title은 최종 결과 제목이다.
```

체크 항목:

```text
input.title을 무조건 필수 검사하지 않는다.
titleProvided=false이면 Gemini가 제목을 생성하도록 프롬프트에 명시한다.
titleProvided=false일 때 sourceTitle fallback을 최종 제목으로 쓰지 않는다.
callback에 processorResult.title을 포함한다.
```

## 7. 입력 정리 노드 기준

입력 정리 노드는 다음 역할만 수행한다.

```text
Gateway payload 파싱
inputType 확인
text/file/audio/image 구분
titleProvided/titleSource 정리
파일 메타데이터 정리
processorInput 생성
```

입력 정리 노드가 하지 말아야 할 것:

```text
권한 판단
회사 설정 조회
개인 설정 조회
보관 레벨 재계산
Firestore 업데이트
```

---

## 8. processorResult 생성 기준

Level 2 이상을 지원하는 워크플로우는 가능한 한 다음 구조를 생성한다.

```json
{
  "title": "결과 제목",
  "summary": "짧은 요약",
  "content": "본문 텍스트",
  "mdContent": "마크다운 본문",
  "hashtags": [],
  "attachments": [],
  "structuredData": {},
  "warnings": []
}
```

Level 1 notify_only도 이메일 전송을 위해 내부적으로 MD 본문을 만들 수 있다. 단, callback에서 processorResult 전체 저장을 요구하지 않아도 된다.

---

## 9. retentionPolicy Router 기준

n8n은 Gateway가 전달한 retentionPolicy를 기준으로 분기한다.

### 9.1 notify_only

```text
이메일 본문 또는 MD, 기타파일 첨부파일 전송 가능
N8Lient DB에 processorResult 전체 저장하지 않음
originalFileRefs/resultRefs callback 포함하지 않음
Storage 업로드 없음
```

### 9.2 processed_result

```text
processorResult callback 포함
원본 파일 참조 제외
resultRefs 기본 제외
MD 다운로드는 앱에서 mdContent 기반 동적 생성
```

### 9.3 full_archive

```text
processorResult callback 포함
originalFileRefs 포함
resultRefs 포함 가능
원본/결과 파일 Storage 또는 export 처리
```

---

## 10. 이메일 보고 기준

이메일 보고는 retentionPolicy와 settings에 따라 수행한다.

가능한 방식:

```text
이메일 본문
MD 첨부파일
원본 첨부파일 첨부
결과 링크 안내
```

주의:

```text
Level 1에서 이메일 MD 첨부는 가능하다.
단, 이 파일은 N8Lient Storage 보관 대상이 아니다.
Level 1에서 앱 재다운로드 링크를 만들면 안 된다.
```

---

## 11. Optional Export 기준

Google Drive 등 외부 내보내기는 Optional Export다.

체크 항목:

```text
optionalExportProvider 확인
none이면 export 생략
google_drive이면 필수 folderId 확인
MD 파일과 첨부파일 폴더 분리
resultRefs 타입 구분
```

Optional Export 실패는 원칙적으로 processorResult 생성 자체를 실패시키지 않는다. 단, workflow 정책에 따라 warning으로 callback에 포함할 수 있다.

---

## 12. callback 기준

n8n은 처리 완료 후 Gateway callback을 호출한다.

성공 callback:

```json
{
  "submissionId": "sub_xxx",
  "status": "success",
  "processorResult": {},
  "originalFileRefs": [],
  "resultRefs": [],
  "warnings": []
}
```

실패 callback:

```json
{
  "submissionId": "sub_xxx",
  "status": "failed",
  "error": {
    "code": "PROCESSOR_FAILED",
    "message": "처리 실패 요약"
  }
}
```

callback에는 Secret, Token, Credential, 원본 파일 내용 자체를 포함하지 않는다.

## 13. 전환 완료 검증

최소 테스트:

```text
titleRequired=false + 제목 없음 + audio 있음 실행 성공
input.title=null 확인
titleProvided=false 확인
processorResult.title 생성 확인
callback success 확인
Level 1 이메일 MD 첨부 가능 여부 확인
Level 2 processorResult 저장 확인
Level 3 originalFileRefs/resultRefs 확인
Optional Export none/google_drive 분기 확인
```

개발 검증:

```text
npx tsc --noEmit
npm run build
Gateway build
n8n 수동 실행 테스트
E2E 실행 테스트
```

## 14. 금지 체크리스트

```text
[ ] input.title을 항상 필수로 검사하지 않는다.
[ ] n8n에서 Firestore를 직접 수정하지 않는다.
[ ] settings를 n8n에서 직접 조회하지 않는다.
[ ] Credential/Token을 settings로 받지 않는다.
[ ] Google Drive를 기본 저장소로 가정하지 않는다.
[ ] retentionPolicy 없이 원본 파일을 무조건 저장하지 않는다.
[ ] callback 없이 결과를 외부에만 남기지 않는다.
```
