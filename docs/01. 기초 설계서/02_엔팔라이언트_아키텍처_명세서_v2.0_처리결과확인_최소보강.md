## 1. 문서 목적

본 문서는 엔팔라이언트 솔루션 헌법의 원칙을 실제 시스템 구조로 해석한 아키텍처 기준 문서다.

본 문서는 세부 구현 코드가 아니라 다음을 정의한다.

```text
서비스 계층
주요 데이터 흐름
권한 경계
Gateway와 n8n 책임 분리
결과 보관 구조
DB/Storage/Optional Export 역할
```

---

## 2. 전체 구조

엔팔라이언트는 다음 계층으로 구성된다.

```text
사용자 / 회사관리자 / 오퍼레이터
        ↓
n8lient-app
        ↓
n8lient-gateway
        ↓
n8lient-workflows(n8n)
        ↓
외부 서비스(Gemini, Gmail, Google Drive 등)

n8lient-app / gateway
        ↓
Firestore / Firebase Storage
```

핵심 호출 원칙:

```text
브라우저는 n8n을 직접 호출하지 않는다.
브라우저는 Gateway 또는 앱 API를 통해 실행한다.
Gateway가 n8n을 서버 간 호출한다.
n8n은 처리 결과를 Gateway callback으로 반환한다.
```

## 3. 계층별 책임

### 3.1 n8lient-app

담당:

```text
로그인
회사 가입 요청
권한별 콘솔
워크플로우 마스터 관리
고객사 계약/설정 관리
사용자 실행 화면
결과 목록/상세/검색 UI
```

n8lient-app은 사용자의 입력을 수집하고 Gateway 실행 요청을 만든다.

### 3.2 n8lient-gateway

담당:

```text
Firebase ID Token 검증
사용자 승인 상태 확인
clientId/role 확인
워크플로우 사용 가능 여부 확인
회사 설정 조회
개인 설정 조회
설정 병합
retentionPolicy 계산
submission 생성
파일 저장/임시 처리
canonicalPayload 생성
n8n 호출
callback 수신
결과 저장
```

Gateway는 실행 정책의 최종 방어선이다.

### 3.3 n8lient-workflows

n8n 워크플로우다.

담당:

```text
Gateway 요청 수신
Header Auth 검증
입력 정리
프로세서 실행
processorResult 생성
retentionPolicy에 따른 결과 후처리
Optional Export 수행
Gateway callback 호출
```

금지:

```text
Firestore 직접 수정
회사/개인 설정 직접 조회
사용자 권한 판단
Gateway를 우회한 결과 저장
```

### 3.4 Firestore

담당:

```text
사용자/회사/계약/설정
워크플로우 마스터
실행 로그
processorResult
검색용 메타데이터
```

### 3.5 Firebase Storage

담당:

```text
Level 3 원본 파일
Level 3 결과 파일
다운로드 대상 파일
```

### 3.6 Optional Export

Google Drive 등 외부 내보내기 기능이다.

원칙:

```text
기본 저장소가 아니다.
retentionPolicy를 대체하지 않는다.
사용자가 선택한 외부 복사/활용 기능이다.
```

## 6. 실행 흐름

표준 실행 흐름:

```text
1. 사용자가 앱에서 자동화 선택
2. 앱이 입력값과 파일을 수집
3. Gateway 실행 요청
4. Gateway가 인증/권한/계약/설정 검증
5. Gateway가 retentionPolicy 계산
6. Gateway가 submission 생성
7. Gateway가 파일을 레벨에 따라 임시 처리 또는 Storage 저장
8. Gateway가 canonicalPayload를 만들어 n8n 호출
9. n8n이 프로세서 실행
10. n8n이 processorResult와 resultRefs를 callback으로 반환
11. Gateway가 submission 업데이트
12. 앱이 결과 목록/상세/검색에 표시
```

## 6-1. 처리 결과 확인 표시 흐름

비동기 실행 완료 후 사용자가 결과 로그를 클릭하면 앱은 결과 상세 화면의 `[02] 처리 결과 확인` 섹션에 워크플로우가 만든 사용자용 결과 메시지와 링크를 표시한다.

```text
1. n8n이 업무 처리 완료
2. n8n이 사용자용 처리 결과 확인 메시지 생성
3. n8n이 result.summary / result.resultUrl / processorResult.summary / actionLinks를 callback에 포함
4. Gateway가 callback을 수신하고 submissions에 저장
5. 앱이 결과 상세 화면에서 [02] 처리 결과 확인 섹션에 표시
```

주의:

```text
- 앱은 워크플로우별 완료 문구를 하드코딩하지 않는다.
- 실행 직후 브라우저가 처리 완료까지 대기하지 않는다.
- 실시간 완료 모달, polling, websocket은 이 계약의 필수 범위가 아니다.
- 사용자는 나중에 결과 목록/최근 실행 로그에서 결과를 열어 확인한다.
```

대표 예:

```text
구글 캘린더 등록 자동화 → 등록 완료 메시지 + 캘린더 일정 링크
문서 생성 자동화 → 생성 완료 메시지 + 문서 보기 링크
영수증 분석 자동화 → 분석 완료 메시지 + 상세 결과 링크
```

## 7. 결과 보관 아키텍처

### 7.1 Level 1 notify_only

```text
N8Lient에는 최소 submission 로그만 남긴다.
결과 본문과 파일은 DB/Storage에 보관하지 않는다.
결과는 이메일 본문, MD 첨부파일, 입력 원본 파일의 이메일 임시 첨부 형태로 전달 가능하다.
```

### 7.2 Level 2 processed_result

```text
Level 1의 이메일 전달 기능 전체를 상속한다.
N8Lient DB에 processorResult를 저장한다.
결과 목록/상세/검색을 제공한다.
MD 다운로드는 DB의 mdContent 기반으로 동적 생성할 수 있다.
원본 파일은 Storage에 보관하지 않는다.
```

### 7.3 Level 3 full_archive

```text
Level 2의 모든 기능에 더해 originalFileRefs/resultRefs를 저장한다.
원본 파일과 결과 파일을 Firebase Storage에 보관한다.
원본 확인, 재검증, 감사 목적에 사용한다.
```

## 8. retentionPolicy 계산 구조

정책 계층:

```text
workflowTemplates.retentionCapabilities
→ clientContracts.contractRetentionLimit
→ clientAutomations.contractRetentionLimit
→ clientAutomations.companyRetentionPolicy
→ userAutomationSettings.userRetentionPreference
→ Gateway 최종 계산
```

선택 가능 레벨:

```text
selectableLevels =
workflowTemplates.retentionCapabilities.supportedLevels
∩ clientAutomations.contractRetentionLimit.allowedLevels
```

실행 선택 우선순위:

```text
userAutomationSettings.userRetentionPreference
→ clientAutomations.companyRetentionPolicy.recommendedLevel
→ workflowTemplates.retentionCapabilities.defaultLevel
→ 시스템 기본값
```

Gateway는 기술 한도와 계약 한도를 초과하는 요청을 보정하거나 차단한다.

## 9. title 계약 아키텍처

실행 제목은 세 가지로 분리한다.

```text
input.title = 사용자가 직접 입력한 제목
submissionTitle = 시스템 임시 제목
displayTitle = UI 표시 제목
processorResult.title = 최종 결과 제목
```

표시 우선순위:

```text
processorResult.title
→ displayTitle
→ submissionTitle
```

`titleRequired=false`인 워크플로우에서는 `input.title=null`이 허용된다.

## 10. n8n 전환 원칙

n8n 워크플로우는 다음 전환 기준을 따른다.

```text
Cloud Memo 등 개인 자동화 의존 제거
Gateway canonicalPayload 기반 입력 정리
settings는 Gateway가 병합한 payload.settings만 사용
retentionPolicy에 따라 결과 분기
Firestore 직접 수정 금지
Google Drive는 Optional Export로만 사용
callback은 Gateway로만 전달
```

## 11. 보안 경계

금지:

```text
n8n 직접 브라우저 호출
Firebase Storage 공개 URL 직접 노출
Firestore에 binary/base64 저장
사용자 설정에 Credential/Token 저장
Google Drive Credential ID를 사용자 settings로 받는 구조
운영 데이터를 테스트 삭제 로직으로 삭제
```

접근 원칙:

```text
개인 사용자 = 자기 submission만 접근
회사관리자 = 자기 clientId의 submission 접근
operator = 운영 목적 접근
```
